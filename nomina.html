2. Nómina/RH

Estructura organizacional

  • Defina sus departamentos como un organigrama sin límite de profundidad.
  • En cada departamento defina tantos puestos como requiera, especificando la naturaleza de la plaza (eventual, permanente, honorarios, asimilados a salarios, becario, etc.) y el número de plazas autorizadas para cada uno.
  • Cada departamento está asociado a un centro de costos para efectos contables y de seguimiento presupeustal.
  • De así requerirse, cada puesto está asociado a un nivel, el cual puede corresponder con mandos superiores, medios o niveles operativos.
  • A un empleado en una plaza, es posible especificarle un rango dentro del nivel al que está asignado su puesto.
  • Un empleado puede ocupar varias plazas.



Tipos de nómina

  • Se pueden definir tantos como se requieran.
  • Pueden definirse por periodicidad de pago, por riesgo de trabajo, por ubicación física, por tipo de personal, etc.
  • En cada tipo de nómina se especifica si va a manejar tabulador o no.
  • Sobre un tipo de nómina se pueden especificar otros tipos, que identificamos como especiales, cuyo propósito es calcular procesos alternativos, como: finiquitos y liquidaciones, vales de despensa, aguinaldo, PTU, etc.
  • Para cada tipo de nómina es posible especificar su formato de recibos de pago. Los formatos de recibos son 100% personalizables.
  • Existen muchos valores que se personalizan para cada tipo de nómina, lo que permite al usuario gran control sobre el comportamiento del sistema.



Tabuladores

  • Para cada tipo de nómina es posible definir tantos conceptos de tabulación como se requieran.
  • Un concepto de tabulación puede estar asociado a un concepto de cálculo.
  • A cada concepto de tabulación se le especifica si aplica para mandos superiores, medios y/o niveles operativos.
  • Para cada uno de los niveles que aplica, cada concepto de tabulación puede tener hasta 10 rangos.



Períodos de cálculo

  • Cada tipo de nómina (ya sea normal o especial, define sus períodos de cálculo según requiera.
  • Puede manejarse cualquier periodicidad de pago:
    • Semanal
    • Catorcenal
    • Quincenal
    • Mensual
    • Bimestral
    • Otros
    • Períodos discontinuos



Tablas

  • Ya sea para pagos de impuestos, destajos, comisiones, etc., el sistema permite definir tablas.
  • Para cada tabla el usuario define el número de columnas que manejará así como los renglones que la integran.
  • Cada tabla especifica la fecha a partir de la cual es vigente. El sistema puede almacenar todas las vigencias de cada tabla.
  • Independientemente de las tablas para cálculos, el sistema permite especificar cualquier cantidad de tablas de prestaciones.
  • Las tablas de prestaciones definen las prestaciones a que tienen derecho los empleados a medida que van cumpliendo años de antigüedad.
  • Al registrar a un empleado en una plaza, también se le especifica su tabla de prestaciones.



Conceptos de cálculo

  • Para cada tipo de nómina que se defina, se especifica el conjunto de conceptos que manejará.
  • El sistema permite especificar conceptos:
    • Percepciones
    • Deducciones
    • Provisiones
    • Estadísticos
  • Para el cálculo de los conceptos el sistema cuenta con Comportamientos por omisión; y cuando se requiere, es posible definir completamente el proceso de cálculo de un concepto en particular.
  • Para el cálculo de la mayoría de los conceptos ya existen procedimientos estándares (incluyendo piramidados), pero si fuera necesario, es posible definir prácticamente cualquier proceso de cálculo que se requiera.
  • Existen relaciones entre los conceptos que permiten llevar a cabo el cálculo de alguno con relación al resultado de cálculo de otros.



Movimientos a empleados

  • El sistema lleva el control de los movimientos de Altas/Bajas/Cambios/Licencias/etc de empleados.
  • Para cada movimiento lleva seguimiento de:
    • Iniciado
    • Requisitado
    • Autorizado
    • Procesado
    • Cancelado
  • De cada empleado lleva todo su historial de movimientos.
  • Adicionalmente se puede llevar el en sistema todo su expediente electrónico, incluyendo la versión digital de sus documentos, foto y una gran cantidad de datos adicionales.
  • Los datos adicionales puede utilizarse para diferentes propósitos, como sus tallas para uniformes, su condición médica, tipo de sangre, familiares, dependientes económicos, idiomas que maneja, otras habilidades, etc.



Incidencias

  • De los conceptos que así se defina, el sistema permite la captura de incidencias.
  • Las incidencias típicamente se utilizan para el registro de los eventos que son diferentes en cada período de pago. Por esta razón también se identifican como Movimientos variables
  • Para cada incidencia el sistema lleva registro de:
    • Fecha del evento.
    • Período en que fue procesado.
    • Comentario
  • En cada incidencia se puede registar ya sea una cantidad y/o un importe. Adicionalmente el sistema permite especificar la parte exenta y gravable, así como otros 2 importes, los cuales pueden servir para diferentes propósitos
  • Las incidencias se pueden incorporar de manera automática desde otros módulos (p.ejm. Asistencia). El sistema también cuenta con funciones de importación desde archivos .csv



Incidencias recurrentes

  • Existen ciertos movimientos variables que son relativamente constantes para un determinado empleado.
  • Por ejemplo, pensiones alimenticias, créditos hipotecarios, FONACTO, préstamos personales, etc.
  • Para estos casos el sistema maneja los Conceptos Fijos.
  • El sistema permite capturar importes y/o porcentajes, así como saldos y otros datos adicionales que ayudan en el cálculo y seguimiento de este tipo de incidencias.
  • Al igual que con el resto de los conceptos, para los Conceptos Fijos el sistema tiene definidos ciertos comportamientos por omisión, pero en caso de ser necesario es posible definir procesos de cálculo particulares.



Proceso de cálculo

  • Es posible procesar una nómina completa o sólo de ciertos empleados, e incluso es posible calcularla para sólo un empleado.
  • En tanto no se cierre el período, el proceso puede repetirse las veces que sea necesario.
  • El resultado del cálculo a detalle de todos los períodos se mantiene almacenado por lo que es posible consultar el detalle del cálculo de cualquier empleado en cualquier período.
  • En tanto un período esté abierto, el resultado puede alterarse, ya sea repitiendo el cálculo o alterando directamente los resultados de los conceptos (sólo usuarios con los privilegios suficientes.
  • Al igual que con el resto de los conceptos, para los Conceptos Fijos el sistema tiene definidos ciertos comportamientos por omisión, pero en caso de ser necesario es posible definir procesos de cálculo particulares.



Informes

  • El sistema cuenta con una muy amplia variedad de informes. Existe informes del período, de acumulados del mes, de acumulados anuales, etc.
  • Prácticamente todos los informes del sistema se obtienen directamente como hojas de Excel.
  • Ciertos informes que así lo requieren salen como archivos PDF, como el diario de la nómina y los recibos fiscales
  • Las ventanas de algunos informes muestran el detalle de la información, por lo que en ocasiones podría ni siquiera ser necesario enviar el informe a Excel.
  • Al igual que con el resto de los conceptos, para los Conceptos Fijos el sistema tiene definidos ciertos comportamientos por omisión, pero en caso de ser necesario es posible definir procesos de cálculo particulares.



Timbrado de recibos

  • El sistema tiene toda la funcionalidad para el timbrado de recibos, por lo que no es necesario ningún tipo de preparación ni envío y recepción de archivos.
  • Una vez que se ha cerrado el período, es posible llevar a cabo el timbrado de los recibos literamente haciendo clic en un solo botón.
  • El sistema lleva el seguimiento de cada recibo, incluyendo su posible cancelación.
  • De manera general, en el precio que se paga por el uso del sistema ya se incluye el timbrado de los recibos.
  • Al igual que con el resto de los recibos, la versión imprimible de los recibos timbrados es totalmente configurable por el usuario.



Sitio Web para Empleados

  • Existe un sitio Web desde el que los empleados tienen acceso a sus recibos y pueden enviarse por correo los correspondientes archivos XML y PDF.
  • De manera estándar se permite la descarga de los últimos 5 años.
  • Incluye hasta el último recibo timbrado sin necesidad de preparar ningún archivo ni enviarlo a ningún servidor web, ni nada por el estilo.
  • Si se requiere, es posible incorporar esta funcionalidad en las páginas web corporativas.
  • El sistema cuenta con funciones que le permiten a los administradores saber qué empleados han entrado al sitio web, así como el número de veces que han bajado cada uno de sus recibos.
  • Los administradores pueden definir qué empleados tienen acceso al sitio web y cuáles no.
  • El sistema también permite el envío por correo electrónico de los recibos directamente simplemente haciendo clic en un sólo botón.



Otros módulos

  • El sistema cuenta con una muy amplia variedad de módulos adicionales.
  • Tenemos módulos como:
    • Control de Asistencia
    • Vacaciones
    • Prestaciones
    • Comedor
    • Presupuesto
    • Otros
  • El grado de complejidad de cada módulo es muy variable, sin embargo, en caso de que se necesite cualquier ajuste o desarrollo, podemos hacerlo.

Condiciones comerciales

Se realiza un pago único por la puesta en marcha y se compran paquetes de Empleados/Procesos, los cuales ya incluyen el proceso de timbrado.

El precio de los paquetes de Empleados/Procesos va disminuyendo a medida que se compre un paquete de mayor tamaño.

Un Empleado/Proceso es un cálculo de nómina para un empleado. Por ejemplo, si se tuviera una nómina quincenal en la que se pagasen las 24 quincenas del año y un cálculo anual de PTU y otro de Aguinaldo, en total serían 26 procesos de cálculo, de manera que, para 100 empleados, esto equivaldría a 2,600 Empleados/Procesos.
